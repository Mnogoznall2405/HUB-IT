## Executive Summary

Проект собирается: `WEB-itinvent/frontend` проходит `npm run build`. Основной риск не в текущей компиляции, а в production-hardening и границах владения: в репозитории есть общий ключ агента, дефолтные учетные записи, небезопасные auth/TLS defaults, XSS-поверхность в почте и несколько крупных модулей, которые уже стали центрами изменений.

Самые срочные зоны:

1. Убрать общий ключ агента и fallback на него из runtime.
2. Сделать auth defaults fail-closed: JWT secret, secure cookies, refresh runtime store, дефолтные пользователи.
3. Закрыть VCS-пароль: не отдавать расшифрованный пароль роли `vcs.read` и не гонять его через query string.
4. Обновить frontend dependencies с уязвимостями: DOMPurify, react-quill/lodash, Vite/Rollup/PostCSS, axios.
5. Зафиксировать стратегию хранилищ и миграций: Alembic/app-db против startup DDL/SQLite/JSON compatibility.

## Architectural Mental Model

Репозиторий сейчас является монорепо HUB-IT:

- `WEB-itinvent/backend` - FastAPI backend, app-owned PostgreSQL/SQLite compatibility, SQL Server inventory integration, auth, chat, mail, KB, VCS, settings.
- `WEB-itinvent/frontend` - React 18/Vite/MUI SPA с крупными route-level страницами.
- `bot` - Telegram bot/legacy domain.
- `agent.py`, `agent/`, `scan_agent/` - Windows inventory/scanning agents and packaging.
- `scan_server/` - scan server with SQLite-backed queues, findings, incidents and worker state.

Главная архитектурная особенность: данные живут сразу в нескольких слоях: SQL Server inventory, app-db, service-local SQLite stores, JSON compatibility manager, env files and agent queues. Это объясняет часть сложности, но также создает риск расхождения схем, прав доступа и runtime state.
