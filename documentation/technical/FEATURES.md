# IT-Invent Bot - Описание функций

## Основные возможности

### 1. Поиск оборудования

#### 1.1 Поиск по серийному номеру
- **OCR распознавание** - автоматическое распознавание серийных номеров с фотографий
- **Ручной ввод** - возможность ввести серийный номер текстом
- **Кэширование** - быстрый повторный поиск (TTL: 2 минуты)
- **Детальная информация** - полные данные об оборудовании из БД

#### 1.2 Поиск по сотруднику
- **Нечеткий поиск** - находит по частичному совпадению ФИО
- **Подсказки** - автоматические подсказки из БД (минимум 2 символа)
- **Пагинация** - удобный просмотр больших списков (5 единиц на страницу)
- **Группировка** - оборудование группируется по типам

#### 1.3 Определение цвета принтера
- **Автоматическое определение** - по модели из БД
- **Ключевые слова** - анализ названия модели
- **Отображение** - цветной 🎨 или ч/б ⚫ в результатах

---

### 2. Перемещение оборудования с актами

#### 2.1 Множественные акты
- **Автоматическая группировка** - по текущим владельцам
- **Отдельный акт** - для каждого старого сотрудника
- **До 10 актов** - за одну операцию
- **Транслитерация** - корректные имена файлов (кириллица → латиница)

#### 2.2 Распознавание серийных номеров
- **OpenRouter API** - высокое качество OCR
- **До 10 фотографий** - за раз
- **Параллельная обработка** - быстрое распознавание
- **Фильтрация** - только оборудование из БД

#### 2.3 Генерация PDF-актов
- **DOCX шаблон** - templates/docx_transfer_act.docx
- **Автоматическое заполнение** - все данные из БД
- **Конвертация в PDF** - через docx2pdf
- **Уникальные имена** - с датой и временем

#### 2.4 Отправка на email

**Автоматическая отправка старым владельцам:**
- Email из таблицы OWNERS
- Персонализированные письма
- Каждому свой акт
- Детальный отчет о результатах

**Ручной ввод email:**
- Все акты одним письмом
- Множественные получатели
- Валидация email адресов

---

### 3. Управление базами данных

#### 3.1 Множественные БД
- **ITINVENT** - основная база
- **MSK-ITINVENT** - Москва
- **OBJ-ITINVENT** - Объекты
- **SPB-ITINVENT** - Санкт-Петербург

#### 3.2 Переключение БД
- **Сохранение выбора** - между сессиями
- **Закрепленное сообщение** - текущая БД всегда видна
- **Статистика** - количество оборудования и сотрудников

#### 3.3 Просмотр оборудования
- **По типам** - МФУ, ноутбуки, мониторы и т.д.
- **По филиалам** - фильтрация по локациям
- **Пагинация** - удобный просмотр
- **Детальная информация** - по каждой единице

---

### 4. Экспорт данных

#### 4.1 Ненайденное оборудование
- **Форматы**: Excel (.xlsx), CSV
- **Фильтры**: дата, БД, только новые
- **Инкрементальный экспорт** - только новые записи
- **Отправка на email** - с вложениями

#### 4.2 Перемещения оборудования
- **Форматы**: Excel, CSV, TXT
- **История** - все перемещения с датами
- **Пути к актам** - ссылки на PDF-файлы
- **Фильтрация** - по дате и БД

#### 4.3 Оборудование сотрудника
- **Полный список** - всё оборудование сотрудника
- **Экспорт в Excel** - с детальной информацией
- **Отправка на email** - прямо из бота

#### 4.4 Замена комплектующих МФУ
- **Периоды**: 1 месяц, 3 месяца, весь период
- **Разделение по филиалам** - отдельные листы Excel для каждого филиала
- **Сводный лист** - агрегированная статистика по всем филиалам
- **Совместимые модели** - колонка с моделями для каждого компонента

#### 4.5 Замена батареи ИБП
- **Периоды**: 1 месяц, 3 месяца, весь период
- **Разделение по филиалам** - отдельные листы Excel для каждого филиала
- **Сводный лист** - агрегированная статистика по всем филиалам
- **OCR данные** - серийный номер, модель, филиал, локация, сотрудник

#### 4.6 Экспорт баз данных
- **Полный экспорт** - всё оборудование из выбранной базы данных
- **Группировка по типам** - отдельные листы Excel для каждого типа оборудования
- **Умные колонки** - динамическая ширина в зависимости от содержимого
- **Счётчик записей** - отображение количества экспортированных записей

#### 4.7 Чистка ПК
- **Периоды**: 1 месяц, 3 месяца, весь период
- **Разделение по филиалам** - отдельные листы Excel для каждого филиала
- **Сводный лист** - агрегированная статистика по всем филиалам
- **OCR данные** - серийный номер, модель, филиал, локация, сотрудник
- **История чисток** - информация о дате последней чистки и общем количестве

---

### 5. Ненайденное оборудование

#### 5.1 Добавление информации
- **Пошаговый ввод** - все необходимые поля
- **Подсказки** - для типа, модели, сотрудника
- **Валидация** - проверка корректности данных
- **Сохранение в JSON** - для последующего экспорта

#### 5.2 Поля для ввода
- Серийный номер
- Тип оборудования (с подсказками)
- Модель (с подсказками)
- ФИО сотрудника (с подсказками)
- Локация
- Инвентарный номер
- IP-адрес
- Статус
- Филиал
- Компания

---

### 6. Регистрация работ

#### 6.1 Замена комплектующих МФУ
- **LLM-детекция** - автоматическое определение доступных компонентов
- **Типы компонентов** - картриджи, фьюзеры, фотобарабаны
- **Цветная детекция** - определение цветных/чб принтеров
- **Совместимые модели** - автоматическое определение номеров комплектующих
- **Кэширование** - оптимизация LLM-вызовов на 60-80%

#### 6.2 Замена батареи ИБП
- **OCR-распознавание** - автоматическое распознавание серийного номера с фото
- **Ручной ввод** - возможность ввести серийный номер вручную
- **Поиск в SQL** - автоматический поиск ИБП по серийному номеру
- **Подтверждение** - отображение полной информации (сертийный номер, модель, филиал, локация, сотрудник)
- **Сохранение в JSON** - запись данных с меткой времени и названием БД
- **Структурированный экспорт** - Excel с разделением по филиалам

#### 6.3 Чистка ПК
- **OCR-распознавание** - автоматическое распознавание серийного номера с фото ПК
- **Ручной ввод** - возможность ввести серийный номер вручную
- **Поиск в SQL** - автоматический поиск ПК по серийному номеру
- **История чисток** - отображение даты последней чистки из JSON
- **Относительное время** - красивое отображение (сегодня, вчера, X дн. назад, X нед. назад, X мес. назад, X г. назад)
- **Визуальное оформление** - разделитель "🕒 ИСТОРИЯ ЧИСТОК"
- **Счетчик чисток** - отображение общего количества чисток
- **Сохранение в JSON** - запись данных с меткой времени и названием БД
- **Структурированный экспорт** - Excel с разделением по филиалам

---

### 7. Система подсказок

#### 6.1 Подсказки сотрудников
- **Источник**: таблицы ITEMS и OWNERS
- **Fallback**: если в ITEMS не найдено, ищет в OWNERS
- **Сортировка**: сначала начинающиеся с запроса, затем содержащие
- **Лимит**: 8 подсказок

#### 6.2 Подсказки моделей
- **Источник**: таблица CI_MODELS
- **Поиск**: по названию модели
- **Сортировка**: по релевантности
- **Лимит**: 8 подсказок

#### 6.3 Подсказки типов оборудования
- **Источник**: таблица CI_TYPES
- **Фильтрация**: по запросу пользователя
- **Лимит**: 15 типов

#### 6.4 Подсказки локаций
- **Источник**: таблица LOCATIONS (с fallback на ITEMS)
- **Поиск**: по описанию локации
- **Лимит**: 8 локаций

---

### 8. Кэширование

#### 8.1 Типы кэшей
- **user_access_cache** - проверки доступа (TTL: 1 час)
- **equipment_cache** - данные оборудования (TTL: 2 минуты)
- **image_analysis_cache** - результаты OCR (TTL: 10 минут)
- **printer_component_cache** - результаты LLM-детекции компонентов (без TTL)

#### 8.2 Преимущества
- Снижение нагрузки на БД
- Быстрый повторный поиск
- Экономия API-запросов к OpenRouter

---

### 9. Контроль доступа

#### 9.1 Механизмы
- **Белый список пользователей** - по Telegram ID
- **Проверка группы** - членство в разрешенной группе
- **Кэширование проверок** - для производительности

#### 9.2 Обработка
- Информативные сообщения об отказе
- Логирование попыток доступа
- Таймауты при проверке

## Технические особенности

### Асинхронная архитектура
- Все операции неблокирующие
- Параллельная обработка фотографий
- Быстрый отклик бота

### Обработка ошибок
- Graceful degradation при недоступности таблиц
- Fallback-запросы к БД
- Детальное логирование

### Валидация данных
- Проверка серийных номеров
- Валидация email адресов
- Защита от SQL-инъекций
- Проверка IP-адресов

### Логирование
- Детальные логи всех операций
- Уровни: DEBUG, INFO, WARNING, ERROR
- Файл: bot.log

---

## Ограничения

- **Максимум 10 фотографий** за одну операцию перемещения
- **Максимум 10 актов** за одну операцию
- **Размер файла** для OCR ограничен Telegram API
- **SMTP-сервер** должен быть доступен для отправки email
- **Email в БД** должен быть указан для автоматической отправки

---

## Требования

### Программное обеспечение
- Python 3.8+
- SQL Server (ODBC Driver)
- Microsoft Word (для генерации PDF) или LibreOffice

### API и сервисы
- Telegram Bot API
- OpenRouter API (для OCR)
- SMTP-сервер (для email)

### База данных
- Таблицы: ITEMS, OWNERS, CI_TYPES, CI_MODELS, VENDORS
- Опционально: LOCATIONS, BRANCHES, STATUS
- Поле OWNER_EMAIL в таблице OWNERS (для автоотправки)
